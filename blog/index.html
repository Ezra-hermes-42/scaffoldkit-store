Stop Copy-Pasting Boilerplate: One Command to Production-Ready Projects

By Ezra Hermes · May 27, 2026 · 5 min read

Every developer knows the ritual. You start a new Python project, and before you write a single line of business logic, you spend 15 to 30 minutes setting up the same boilerplate you've set up a hundred times before:

  • Creating the project directory structure
  • Writing a Dockerfile and docker-compose.yml
  • Setting up a GitHub Actions CI/CD pipeline
  • Configuring flake8, pytest, and a Makefile
  • Copying a .gitignore and .editorconfig from an old project
  • Writing a README template

It's tedious, error-prone, and — worst of all — completely unnecessary.

Enter ScaffoldKit: a CLI tool that generates production-ready Python, Node.js, and Go projects in a single command. This tutorial walks you through generating a Python project, running the tests, and understanding exactly what you get — so you can stop copy-pasting and start shipping.

What You'll Need

  • ScaffoldKit — purchase and download here
  • Python 3.8+ (for the generated project, not the CLI — the CLI itself has no dependencies)
  • Docker (optional, for containerized development)

Step 1: Generate a Project

Open your terminal and run:

$ scaffoldkit python my-app --all

The --all flag tells ScaffoldKit to include every optional module: Docker, Docker Compose, GitHub Actions CI/CD, linting (flake8), testing (pytest with coverage), a Makefile, .editorconfig, .gitignore, and a README template.

Within milliseconds, a fully structured project appears in a directory called my-app/.

Every file is production-ready from the start:

  • app/main.py — A FastAPI application with a health-check endpoint, proper error handling, and logging configured.
  • Dockerfile — Multi-stage build that installs dependencies, runs tests, and produces a slim production image.
  • docker-compose.yml — Hot-reload development setup with volume mounts.
  • .github/workflows/ci.yml — Matrix CI across Python 3.8, 3.9, 3.10, 3.11, and 3.12, running linting, tests, build verification, and coverage.
  • Makefile — Common targets: make install, make test, make lint, make build, make run.
  • .flake8 — Sensible defaults already configured (line length 88, compatible with Black).

Beyond Python: Node.js and Go

The same --all flag works for Node.js and Go projects too:

# Generate a Node.js project with everything
$ scaffoldkit node my-api --all

# Generate a Go project with everything
$ scaffoldkit go my-service --all

Each stack follows the same conventions — the same Makefile targets, Docker patterns, and CI structure — so switching between stacks feels natural.
